Where it stands, honestly

What shipped is a well-structured reading surface: collision-proof /[city]/[id]-[slug] routing, the hero bento, feed, sticky rail, and the warm editorial design system, all reading the live hr_* tables through the RLS wall. What is not built is most of what makes this a publication rather than a template: there is no way for a human to author content, article bodies are never actually rendered, there are no accounts, no images, no discovery surface, and no ad or analytics machinery. The 12 seeded stories all carry placeholder bodies.

Two hard blockers to unblock before P0 can finish

1 · Supabase Postgres password / connection string. Payload CMS connects directly to Postgres via @payloadcms/db-postgres. Schema work can go through the Supabase MCP channel, but the running /admin needs the real DATABASE_URI.

2 · Resend (or Mailchimp) API key. The newsletter writes rows today but cannot send a confirmation or a dispatch without an ESP credential.

Neither blocks rendering + auth work, which can start immediately
